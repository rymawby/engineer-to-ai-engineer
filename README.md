# Engineer to AI Engineer

I recently made the switch from being classed as an "Engineer" to an "AI Engineer". A lot of things are the same, and a lot of things are different. Basically, there are a lot of things.

To make that switch, I had to learn a lot of AI-specific concepts that I was not using in my day job. I know a lot of people are in the same boat: either trying to make that move themselves, or trying to future-proof the engineering role they already have.

This repo is a GitHub-friendly version of the revision crib sheet I put together while learning those concepts.

## What this covers

The notes focus on practical AI engineering and agent-system concepts:

- LLMs, embeddings, vector spaces, and stochastic systems
- RAG, retrieval, chunking, and vector databases
- Agents, tool use, MCP servers, and production guardrails
- Distributed-systems concepts that matter for AI workflows
- Evaluation, observability, ROI, safety, and senior-level system design

## How to use this repo

Start with the [revision index](docs/README.md), then work through the files in order. Each topic is intentionally short and interview-oriented: learn the concept, understand the trade-offs, and practise explaining when you would or would not use it.
