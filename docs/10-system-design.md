# System Design

AI system design is about choosing where knowledge lives, where actions happen, and where humans or deterministic systems need control.

## Concepts To Know

- RAG assistants
- Customer support agents
- Refund agents
- Knowledge vs actions
- Human-in-the-loop systems

## Revision Prompts

- How would you design a RAG assistant over private documents?
- How would a customer support agent retrieve policy and account context?
- Why is a refund agent riskier than a knowledge assistant?
- How do knowledge tasks differ from action tasks?
- Where would you add human approval?

## Interview Angle

Separate answering from acting. Reading knowledge is lower risk than changing state, moving money, issuing refunds, deleting data, or contacting customers.
