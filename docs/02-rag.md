# RAG

Retrieval-augmented generation, or RAG, lets a system fetch relevant external context before asking an LLM to answer.

- What information problem does RAG solve?
- When would you choose RAG instead of fine-tuning?
- What are the stages of a basic RAG pipeline?
- Why can RAG reduce hallucinations without eliminating them?
- When would RAG be unnecessary or the wrong architecture?

## Interview Angle

RAG is usually about grounding responses in current or private knowledge. Fine-tuning is usually about changing model behaviour, tone, format, or task performance. They solve different problems.
