# Observability

Observability makes AI systems debuggable. You need to see prompts, retrieval, tool calls, model responses, latency, costs, and failure patterns.

## Concepts To Know

- Tracing
- Tool call logs
- Latency breakdowns
- Detecting loops and inefficiencies

## Revision Prompts

- What should a trace show for a RAG or agent request?
- What should be logged about tool calls?
- How would you break down latency across retrieval, model calls, and tools?
- How can you detect repeated loops or wasteful reasoning?

## Interview Angle

If you cannot inspect the system path, you cannot confidently improve it. AI observability needs both model-level and normal application-level telemetry.
